	\section{Randomized Algorithms: Motivating Problems}

Randomized algorithms are widely used for processing large datasets and strings. Their power comes from trading exact deterministic behavior for efficiency, while still guaranteeing correctness with high probability.

We now study two representative settings:
\begin{itemize}
    \item randomized computation models (Monte Carlo vs Las Vegas),
    \item randomized string processing via hashing (fingerprinting).
\end{itemize}

\subsection{Monte Carlo and Las Vegas Algorithms}

\vspace{0.5em}

\textbf{Monte Carlo Algorithms.}
A Monte Carlo algorithm has deterministic runtime but may output \hide{an error} with small probability.

Formally, for input $x$, it outputs $A(x)$ such that
\[
\Pr[A(x) \neq f(x)] \le \delta.
\]

\vspace{0.3in}

\textbf{Las Vegas Algorithms.}
A Las Vegas algorithm always outputs the correct answer, but has random runtime.

\[
\Pr[A(x) = f(x)] = 1,
\]
but the runtime is a \hide{random variable}.

\vspace{0.5in}

\textbf{One-sided Error.}
A Monte Carlo algorithm has one-sided error if it can only fail in one direction (e.g., false positives), i.e., \hide{asymmetric failure}. 

\newpage

\begin{Qu}
Suppose a Monte Carlo algorithm has error probability $1/3$. If we run it $k$ independent times and take majority vote, what happens to the probability of success?

\begin{itemize}
    \aitem It remains $2/3$
    \bitem It approaches $1$ as $k$ increases
    \citem It approaches $0$ as $k$ increases
    \ditem It becomes exactly $1$ after $k = 3$
\end{itemize}
\end{Qu}

\vspace{0.5in}

\textbf{Probability Amplification.}
Repeating an algorithm reduces failure probability to \hide{exponentially small} in the number of repetitions, i.e., $\Pr[\text{failure}] \le e^{-\Omega(k)}$.

\vspace{1em}

\subsection{Fingerprinting and String Matching}

We study the string matching problem: given pattern $P$ and text $T$, find all occurrences of $P$ in $T$. 
A na\"{i}ve algorithm compares each alignment directly, giving runtime $\hide{O(nm)}$.

\vspace{1em}

Instead, we use \hide{fingerprinting}, where strings are mapped to compact hash values.

A hash function:
\[
h: \Sigma^m \to \mathbb{Z}_M
\]
assigns each string a numeric fingerprint.

We compute:
\begin{itemize}
    \item $h(P)$ for the pattern,
    \item $h(T[i:i+m])$ for substrings of $T$.
\end{itemize}

\vspace{0.5em}

Key property:
\[
P = Q \implies h(P) = h(Q),
\]
but equality of hashes does not guarantee equality of strings due to \hide{collisions}.

\paragraph{Rolling Hashes}

Directly computing each substring hash costs $O(m)$ per position. The goal is to update hashes in \hide{constant time} as the window shifts.

\newpage

\subsection{The Karp--Rabin Algorithm}

We define a rolling hash using base $b$ and modulus $M$:
\[
h(S) = \left(\sum_{i=0}^{m-1} s_i \cdot b^{m-1-i}\right) \bmod M.
\]

This treats strings as \hide{polynomials modulo $M$}.

\vspace{0.5em}

\textbf{Example 1 (numeric alphabet).}
Suppose the alphabet is digits $\{0,1,\dots,9\}$, and consider the string $S = 314$ with base $b = 10$:
\[
\phantom{h(S) = 3 \cdot 10^2 + 1 \cdot 10^1 + 4 \cdot 10^0 = 314.}
\]
If we take modulus $M = 101$, then:
\[
\phantom{h(S) = 314 \bmod 101 = 11.}
\]

\vspace{0.5em}

\textbf{Example 2 (characters).}
Map characters to integers, e.g., $a=1, b=2, \dots, z=26$.  
Let $S = \texttt{cab}$, with $b = 31$:
\[
\phantom{h(S) = 3 \cdot 31^2 + 1 \cdot 31^1 + 2 \cdot 31^0.}
\]
Expanding:
\[
\phantom{h(S) = 3 \cdot 961 + 1 \cdot 31 + 2 = 2883 + 31 + 2 = 2916.}
\]
If $M = 1000$, then:
\[
\phantom{h(S) = 2916 \bmod 1000 = 916.}
\]

\vspace{0.5em}

\textbf{Interpretation.}
The hash is simply evaluating a polynomial at $b$, then reducing modulo $M$.  
Different strings correspond to different polynomials, but distinct polynomials may agree modulo $M$, leading to \hide{collisions}.
\vspace{0.5in}

\textbf{Rolling update rule.}
When shifting the window:
\begin{itemize}
    \item remove left character,
    \item multiply by $b$,
    \item add new character.
\end{itemize}

Thus all substring hashes are computed in total time $\hide{O(m)}$.

\vspace{0.5in}

\textbf{Correctness guarantee.}
If $P$ occurs in $T$, then their hashes match exactly:
\[
h(P) = h(T[i:i+m]).
\]

However, we may still have:
\[
h(P) = h(T[i:i+m]) \quad \text{but} \quad P \neq T[i:i+m],
\]
due to \hide{hash collisions}.

\vspace{0.5em}

The probability of collision can be made \hide{negligible} by choosing large $M$.

\newpage

\section{Application 1: Checking if Two Strings Match}

Fingerprinting gives a randomized method for string equality testing.

\begin{theorem}
If two strings are identical, then their fingerprints match:
\[
x = y \implies h(x) = h(y).
\]
\end{theorem}

\textit{Proof.}
Equality implies identical polynomial representations, hence identical hashes modulo $M$.

\vspace{0.5em}

The converse may fail due to \hide{rare collisions}:
\[
h(x) = h(y) \centernot\implies x = y.
\]

Thus fingerprinting is a Monte Carlo equality test: always correct when strings differ, but may fail due to \hide{collision events}.
	
\section{Application 2: Hash Tables and Dictionaries}

Given a universe of keys $U$, a dictionary maintains a dynamic set of elements supporting insert, delete, and search operations.

\vspace{0.5em}

We resolve collisions using chaining:
\begin{enumerate}
    \item Map each key to a slot using a hash function $h(k)$.
    \item Store all elements that hash to the same slot in a linked list.
\end{enumerate}

\newpage

\begin{theorem}
For any deterministic hash function, an adversary can choose $n$ keys that all hash to the same slot, causing search time $\Theta(n)$.
\end{theorem}

\begin{proof}
Let $U$ be a universe of keys with $|U| > n \cdot m$.

\begin{enumerate}
    \item When hashing $|U|$ keys into $m$ slots, by the pigeonhole principle there exists a slot containing at least
    \[
    \left\lceil \frac{|U|}{m} \right\rceil > n
    \]
    keys. Reason: \hide{pigeonhole principle}

    \item Since the hash function is deterministic, the adversary can identify a subset of $n$ keys that all map to this same slot. \\
		Reason: \hide{adversary knows hash function}

    \item These keys form a linked list of length $n$ under chaining, so searching requires linear scan. \\
		Reason: \hide{cost proportional to list length}
\end{enumerate}

Therefore, the worst-case search time is \hide{$\Theta(n)$}.
\end{proof}

	\section{Universal Hashing and Expected Chain Lengths}
	If $U$ is a universe of keys and $m$ is the table size, a \emph{universal hash family} $\mathcal{H}$ is a collection of hash functions in which the probability of a collision between any two distinct keys is at most $1/m$. \\
	
	Let $x, y \in U$ be distinct keys and assume that $h$ is chosen uniformly at random from $\mathcal{H}$. 
	
	The \emph{expected chain length} for a hash table is bounded as follows:
	\begin{itemize}
		\item There is a random choice of $h \in \mathcal{H}$ at runtime
		\item There is a collision $h(x) = h(y)$ with probability $\le 1/m$
	\end{itemize} 
	
	\vs{5cm}
	
	The \emph{load factor} of a hash table with $n$ elements and $m$ slots is $\alpha$.
	\begin{align*}
		\alpha = \frac{n}{m}
	\end{align*}
	
	\begin{lemma}
		If we use universal hashing, the expected time for a search operation is \hide{$\Theta(1 + \alpha)$ }
	\end{lemma}

\section{Introduction: Web Search and PageRank}

\begin{center}
\includegraphics[width = .75\linewidth]{query-university.png}
\end{center}

Before Google, ranking web pages was largely based on simple text matching.  
The \textbf{PageRank algorithm}, introduced by Larry Page and Sergei Brin, assigns each webpage a score reflecting its \emph{importance} in the web graph.

\vspace{0.5em}

Given webpages labeled $1,\ldots,n$, the PageRank of a page $i$ depends on the pages that link to it. However, we do not treat all incoming links equally.

\vspace{0.5em}

Instead, we weight contributions as follows:
\begin{itemize}
    \item Links from high-PageRank pages contribute \hide{more}.
    \item Links from pages with many outgoing links contribute \hide{less}.
\end{itemize}

\vspace{0.5em}

The first principle is circular, but this is exactly what makes the model powerful.

\newpage
\section{BrokenRank (Almost PageRank)}

Let $G = (V,E)$ be a directed graph of the web.

Define $L_{ij} = 1$ if $j \to i$, and $0$ otherwise.  
Let $m_j = \sum_{k=1}^n L_{kj}$ denote the out-degree of $j$.

\vspace{0.5em}

We define a preliminary (but flawed) version of PageRank. The \textbf{BrokenRank} $p_i$ is
\[\phantom{
p_i = \sum_{j \to i} \frac{p_j}{m_j}
    = \sum_{j=1}^n \frac{L_{ij}}{m_j} \, p_j.}
\]

This reflects our intuition: contributions scale with $p_j$ and are diluted by $m_j$.

\subsection*{BrokenRank as a Markov Chain}

A \textbf{Markov chain} is a random process that moves between states $1,\dots,n$, where each step corresponds to a transition.  
It is described by an $n \times n$ matrix $P$, where
\[
\phantom{P_{ij} = P(\text{go from } i \text{ to } j).}
\]

\vspace{0.5em}

Let $p^{(0)}$ be an initial probability distribution.  
After one step,
\[
\phantom{p^{(1)} = P^T p^{(0)}.}
\]
Each state collects mass from \hide{all others}.

\vspace{0.5em}

Now consider a Markov chain where the states are webpages and the transition matrix is $A^T$.  
Since $(A^T)_{ij} = A_{ji} = L_{ji}/m_i$, we have
\[
\phantom{
P(\text{go from } i \text{ to } j) =
\begin{cases}
\frac{1}{m_i} & \text{if } i \to j, \\
0 & \text{otherwise}.
\end{cases}
}
\]

\vspace{0.5em}

This corresponds to a \emph{random surfer} who follows outgoing links uniformly at random, with probability \hide{$1/d_{\text{out}}(u)$}.

\vspace{0.5em}

\textsc{GenericRandomWalk}$(G)$
\begin{enumerate}
    \item Initialize $u$ to a random page.
    \item While the surfer continues:
    \begin{enumerate}
        \item Pick an outgoing edge $(u,v)$ uniformly at random.
        \item Set $u \leftarrow v$.
    \end{enumerate}
\end{enumerate}

\newpage
\subsection{Why is BrokenRank Broken?}

The issue is that this Markov chain is not \textbf{strongly connected}. This occurs in several scenarios:
\begin{itemize}
    \item \textbf{Disconnected components.}
    \item \textbf{Dangling nodes (dead ends):} reaching such a node causes probability mass to \hide{vanish}.
    \item \textbf{Spider traps:} a set $S \subseteq V$ with no outgoing edges traps the walk once entered; the surfer \hide{cannot escape}.
\end{itemize}

\section{PageRank as a Markov Chain}

\paragraph{Key idea.}
Modify the process so the surfer never gets trapped.

\vspace{0.5em}
Let $E$ be the $n \times n$ all-ones matrix, i.e., $E_{ij} = 1$ for all $i,j$.  
Define
\[
\phantom{
A = \frac{1-d}{n}E + d L M^{-1},}
\]
where $d \in (0,1)$ is the teleportation parameter (typically $0.85$).

Then
\[
P(\text{go from } i \text{ to } j) =
\begin{cases}
\frac{1-d}{n} + \frac{d}{m_i} & \text{if } i \to j, \\
\frac{1-d}{n} & \text{otherwise}.
\end{cases}
\]

\vspace{0.5em}

This is a \textbf{random surfer with teleportation}: with probability $d$ follow a link, otherwise jump uniformly.

\vspace{0.5em}

Teleportation ensures the walk can always \hide{escape traps}, making the chain \textbf{strongly connected}.  
Therefore, the stationary distribution is \textbf{unique}.

\vspace{0.5em}

A \emph{PageRank vector} $\pi$ satisfies
\[
\pi = A^T \pi,
\]
i.e., it is a \hide{stationary distribution}.

\newpage
\subsection{Generic Teleportation Update}

\textsc{GenericPageRankUpdate}$(G,v,d)$
\begin{enumerate}
    \item $jump \gets (1-d)/|V|$
    \item $link\_sum \gets \sum_{(u,v)\in E} d \cdot \frac{\pi(u)}{d_{\text{out}}(u)}$
    \item \textbf{return} $jump + link\_sum$
\end{enumerate}

\begin{lemma}
For any teleportation parameter $d \in (0,1)$, the induced Markov chain is \textbf{strongly connected}. Consequently, the stationary distribution (the PageRank vector) $p$ is \textbf{unique}.
\end{lemma}

\vspace{0.5em}

\paragraph{Why does this matter?}
This lemma is the key guarantee that PageRank is well-defined and computable.  
In contrast, BrokenRank failed due to structural issues in the web graph: \emph{disconnected components}, \emph{dangling nodes}, and \emph{spider traps}. Teleportation resolves all of these simultaneously.

\vspace{0.5em}

\begin{itemize}
    \item \textbf{Disconnected components (strong connectivity).}  
    Since $d < 1$, the teleportation probability $(1-d)$ is strictly positive.  
    From any page, there is a nonzero probability of jumping to \emph{any} other page in a single step.  
    Effectively, teleportation introduces \hide{a complete set of directed edges} between all nodes, making the graph strongly connected.

    \item \textbf{Spider traps and cycles (convergence).}  
    Without teleportation, probability mass can get trapped or cycle indefinitely.  
    Teleportation breaks this behavior by allowing the surfer to leave any set of nodes at any time.  
    In particular, there is always a chance of jumping \hide{back to the current node}, which acts like a self-loop and prevents periodic oscillations.
\end{itemize}

\section{Computing PageRank}

We compute $\pi$ via repeated multiplication by $A$.

\vspace{0.5em}

\textsc{PageRank-Power-Iteration}$(G)$
\begin{enumerate}
    \item Initialize $\pi(u) = 1/|V|$ for all $u \in V$.
    \item Choose teleportation parameter $d$ (typically $0.85$).
    \item Iteratively update scores using incoming edges and teleportation.
    \item Output the values once they stabilize.
\end{enumerate}

\subsection{Power Iteration Details}

We represent the PageRank vector and related quantities as follows:
\begin{itemize}
    \item $\pi \in \mathbb{R}^V$, where $\pi(u)$ is the PageRank score of node $u$.
    \item $d_{\text{out}}(u)$ denotes the out-degree of node $u$.
    \item $\pi_{\text{new}}$ is an auxiliary array used for synchronous updates.
\end{itemize}

\begin{algorithm}
\caption{\textsc{PageRank-Power-Iteration}$(G,d)$}
\begin{algorithmic}
\State Initialize $\pi(u) \gets 1/|V|$ for all $u \in V$
\Repeat
    \State $\pi_{\text{new}}(v) \gets (1-d)/|V| \quad \forall v \in V$
    \For{$(u,v) \in E$}
        \State $\pi_{\text{new}}(v) \gets \pi_{\text{new}}(v) + d \cdot \frac{\pi(u)}{d_{\text{out}}(u)}$
    \EndFor
    \State $\Delta \gets \max_{v \in V} |\pi_{\text{new}}(v) - \pi(v)|$
    \State $\pi \gets \pi_{\text{new}}$
\Until{$\Delta \le \epsilon$}
\State \Return $\pi$
\end{algorithmic}
\end{algorithm}

\end{Qu}
